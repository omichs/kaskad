# Kaskad — каскадный NAT на iptables

Универсальный bash-скрипт для настройки каскадных соединений, переадресации трафика (NAT) и ускорения сети на Linux.

Создаёт «мост» между клиентом и зарубежным VPN/Proxy сервером: клиент подключается к локальному VPS, который прозрачно пересылает трафик дальше — на уровне ядра, без лишних процессов.

```
Клиент  →  [Этот VPS · iptables NAT]  →  Зарубежный VPN/Proxy
```

## Возможности

- **Быстрый запуск** — после установки доступен глобальной командой `gokaskad`
- **Kernel NAT** — работает через iptables, никаких userspace-прокси, скорость ограничена только каналом сервера
- **BBR Turbo** — автоматически включает алгоритм Google BBR для ускорения TCP
- **Мультипротокольность** — UDP (AmneziaWG, WireGuard), TCP (VLESS, VMess, Reality, MTProto)
- **Кастомные правила** — разные порты входа и выхода, проброс SSH, RDP и любых других сервисов
- **Мульти-туннелирование** — несколько правил на разных портах одновременно
- **Умная настройка UFW** — точечное открытие портов без изменения глобальной политики форвардинга
- **Персистентность** — правила сохраняются после перезагрузки через `netfilter-persistent`
- **Валидация ввода** — проверка корректности IPv4-адресов и диапазона портов

## Требования

- ОС: Ubuntu 20.04+ или Debian 10+
- Доступ: `root`
- Тип VPS: KVM или Dedicated (OpenVZ/LXC без поддержки `tun` на хосте — не поддерживается)

## Установка и запуск

```bash
wget -O install.sh https://raw.githubusercontent.com/omichs/kaskad/main/install.sh \
  && chmod +x install.sh \
  && ./install.sh
```


При первом запуске скрипт автоматически регистрируется в системе. Все последующие запуски — одной командой:

```bash
gokaskad
```

## Инструкция: как настроить каскад

### Шаг 1 — Подготовка

Подготовьте данные от вашего **зарубежного** сервера:

- **IP-адрес** (например: `45.10.20.30`)
- **Порт** (на котором работает целевой сервис)

### Шаг 2 — Настройка этого сервера

1. Запустите `gokaskad`
2. Выберите протокол (пункты 1–3) или создайте кастомное правило (пункт 4)
3. Введите IP-адрес зарубежного сервера и порт(ы)
4. Скрипт сообщит об успешном создании «моста»

### Шаг 3 — Настройка клиента

1. Откройте приложение (AmneziaWG, v2rayNG, Telegram и т.д.)
2. В настройках соединения найдите поле **Endpoint / Адрес сервера**
3. Замените зарубежный IP на **IP этого сервера**
4. Сохраните и подключитесь

## Меню

| Пункт | Действие |
|-------|----------|
| `1` | Настроить AmneziaWG / WireGuard (UDP) |
| `2` | Настроить VLESS / XRay (TCP) |
| `3` | Настроить TProxy / MTProto (TCP) |
| `4` | Кастомное правило (разные порты, SSH, RDP...) |
| `5` | Просмотр активных правил |
| `6` | Удалить одно правило |
| `7` | Сбросить все настройки |
| `8` | Инструкция |
| `0` | Выход |

## Как это работает

Скрипт настраивает четыре правила iptables:

```
INPUT      ACCEPT       — открыть входящий порт
PREROUTING DNAT         — подменить dst: перенаправить пакет на зарубежный IP:порт
FORWARD    ACCEPT       — разрешить пересылку трафика
POSTROUTING MASQUERADE  — подменить src: пакеты уходят от имени этого сервера
```

Обратный трафик возвращается зеркально. Всё происходит в ядре Linux — дополнительные процессы не запускаются.

## Известные ограничения

- Только **IPv4** — `ip6tables` не используется
- Только **Ubuntu / Debian** — требуется `apt-get` и `dpkg`
- Пункт **«Сбросить все настройки»** удаляет весь iptables целиком, включая правила Docker, fail2ban и других программ — используйте с осторожностью
